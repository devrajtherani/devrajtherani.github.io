CuttApi

CuttApi is a URL shortener modules for Python. It allows you to easily shorten URLs using the Cuttly URL shortener service just by entering your API key. The module takes care of the rest automatically.

The conceiving of the thought of CuttApi dates back to the summer of 2022. With a newly found interest in Python, the experimenting with APIs and performing web functions in a terminal were the drive to test Cuttly's API.
Developing CuttApi started with the understanding of the status codes of Cuttly's API, and with solving it in a form which creates a streamlined form for users to shorten URLs right from their terminals or a background process.
The easy part was the incorporation of status codes for name, API key vaidation; but URL Validation needed deep, sound research.
For URL Validation, a separate class yurl was developed which used various request status codes to tranform URL into a final form which can be used for validations as well as same link creation redundancies.

Integration

The Integration was created as a low cost, completely efficient initiative to handle everyday quizzing tasks for the quiz club, using :

  • Google Forms
  • Google Sheets
  • Apps Script
  • Microsoft Excel
  • Macros

The integration, a mix of 5 application was a project created to streamline the operations associated with the implementation everyday quizzing in the club.

  • Google Forms - Google Forms closes every night at 11:55 and reopens back at 12:05. Anyone, in the meantime can fill their responses which are then saved in Google Sheets.
  • Google Sheets - Data entered in Forms gets saved in a Master Sheet in Google Sheets. Previous day's data gets transferred to the Master Sheet linked with Excel.
  • Apps Script - Facilitator of closing, updating, and reopening forms; creator of a new sheet and attaching it to Forms; mover of previous day's data to Master Sheet and deleting last day's sheet after detaching from Forms and transferring data.
  • Microsoft Excel - Processor of complete data locally. Linked to Master Sheet in Google Sheets, the sheet fetches its data and processes it based on complex formulaes to derive the correct responders. Master Workbook in Excel stores the data of the last day, fetching it from the main sheet (using button click), in it based on the year. Data of two years back is removed when new year begins.
  • Macros - Apps Script of local segment. Responsible for creation, updation and removal of relevant years. On-click fetches data from main sheet and save it carefully in the relevant sheets and proper checks (redundancy, year)

The project focused on the following points while in development:

  • Easy use
  • Cost Efficiency

How was Usage Tranquilness ensured?

In the Integration, the user has to perform only two things:

  • Input of data:
    • Text part of quetions
    • Options
    • Public link of an image (if required in any question)
  • Saving the data

Only doing these jobs ensures the integration automatically:

  • Takes user input
  • Saves it
  • Processes it
  • Keeps data ready to archive

How was Cost Efficiency ensured?

Using Google Sheets and Excel together helps in the same:

  • Google provides some amount of storage to keep the data. Regularly deleting some data ensures not much kb or mb is used for saving the data on cloud.
  • Linking Google Sheets to Excel provides a podium to store complete data locally and processing it offline to ensure processing does not crash online, and instead of cloud, data is stored in the unit, exploiting user's unit storage, making it an expandable or storage changable server.
